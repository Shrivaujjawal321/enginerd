### 2.5 Why Verbal eats 90 min — and what to do about it

Cognizant **deliberately invests** in verbal because of US client base. RCs are **business case studies, tech industry analyses, leadership essays** — not pure literature. Strategy:

- **First 10 min**: Skim grammar + para-jumble + sentence-correction questions (~10 Qs). Knock these off fast.
- **Next 60 min**: 4 RC passages × 15 min each. **Read passage fully once**, then attack questions sequentially.
- **Last 20 min**: Cloze tests, vocab-in-context, leftover. Aggressive guess on unknowns.

> Don't read RC paragraph-by-paragraph hoping to skim. Cognizant RC questions are **inference-heavy** — main idea + author's tone + implication. Skimming = wrong answer 50% of the time.

### 3.3 Five worked Python solutions — typical Cognizant problems

#### Problem 1: Second Largest Element

> Given an array of N integers, find the second largest distinct element. If it doesn't exist, print -1.

```python
n = int(input())
arr = list(map(int, input().split()))

first = second = float('-inf')
for x in arr:
    if x > first:
        second = first
        first = x
    elif x > second and x != first:
        second = x

print(second if second != float('-inf') else -1)
```

**Time**: O(n), single pass. **Space**: O(1). Edge case: all elements same → second stays -inf → output -1.

#### Problem 2: Check Palindrome (ignoring spaces + case)

> Given a string, check if it's a palindrome ignoring spaces and case. Print "Yes" or "No".

```python
s = input().strip().lower().replace(" ", "")
print("Yes" if s == s[::-1] else "No")
```

**Time**: O(n). **Slick one-liner.** Cognizant loves these — clean code matters for full credit. The `[::-1]` trick reverses the string in O(n) without explicit loop.

#### Problem 3: Frequency Count of Characters

> Given a string, print each character followed by its frequency in alphabetical order.

```python
from collections import Counter

s = input().strip()
c = Counter(s)
for ch in sorted(c):
    print(f"{ch}{c[ch]}")
```

**Time**: O(n log n) due to sort. `Counter` saves 5 lines of manual dict-incrementing. Output format example: `"hello"` → `e1 h1 l2 o1`.

#### Problem 4: Armstrong Number Check

> An Armstrong number is one where sum of cubes of its digits equals itself (for 3-digit). Generalised: sum of (digit^number_of_digits) equals number.

```python
n = int(input())
digits = [int(d) for d in str(n)]
k = len(digits)
total = sum(d**k for d in digits)
print("Yes" if total == n else "No")
```

**Time**: O(k) where k = number of digits. Cognizant's favourite math problem; appears in ~15% of cycles in some form.

#### Problem 5: Reverse Words in a Sentence

> Given a sentence, reverse the order of words but keep words intact. Example: "I love coding" → "coding love I".

```python
s = input().strip()
words = s.split()
print(" ".join(reversed(words)))
```

**Time**: O(n). `split()` + `reversed()` + `join()` — Cognizant grader respects this idiomatic Python. Trap: candidates write nested loops with manual char-reversal — works but loses **clean-code marks** in GenC Next track.

### 3.4 Coding strategy for the 45-min budget

- **Min 0-5**: Read problem + dry-run on visible test cases on paper.
- **Min 5-10**: Identify approach (brute force first, optimise only if N suggests TLE).
- **Min 10-30**: Code the solution.
- **Min 30-40**: Run on visible test cases, debug.
- **Min 40-45**: Submit. Don't keep over-engineering.

> Bhai sun: **Cognizant ke hidden test cases include edge cases like empty input, single-element, all same**. Teri code in cases pe gracefully handle kar le ye verify kar — that's what gets you GenC Next over GenC.

### 4.4 Speaking — scoring rubric (publicly inferred)

| Dimension | Weight | What scorers look for |
|-----------|--------|------------------------|
| **Fluency** | 25% | Smooth flow, no long pauses (>3 sec) |
| **Pronunciation** | 20% | Clear, neutral accent (Indian English fine, mumbled bad) |

### 4.5 Writing — 200-word essay framework

**Structure (memorise this 4-paragraph template)**:

1. **Intro (40 words)**: Restate prompt, state your stance / thesis.
2. **Argument 1 (60 words)**: Strongest point + 1 example.
3. **Argument 2 (60 words)**: Second point + counter-perspective acknowledged.
4. **Conclusion (40 words)**: Summarise stance, end with forward-looking statement.

**Sample prompt + response (200 words)**:

> Prompt: "Will artificial intelligence replace software developers within the next decade?"

> Artificial intelligence is reshaping software development, but it will not fully replace developers within the next decade. Instead, AI will redefine what developers do.

> Tools like GitHub Copilot and ChatGPT have already automated boilerplate code generation, accelerating routine tasks like CRUD APIs and unit-test scaffolding. This shifts developer focus from typing syntax to designing systems, debugging complex logic, and ensuring security. A senior developer using AI is now five times more productive, but the developer is still essential to direct the AI.

> However, AI today struggles with ambiguous requirements, novel architectures, and ethical judgment in production decisions. Software is built for humans by humans who understand business context. Until AI can hold a stakeholder meeting and negotiate trade-offs, full replacement remains distant. The 2024 layoffs at tech firms reflect economic cycles, not AI displacement.

> In conclusion, the next decade will see AI as a productivity multiplier rather than a replacement. Developers who embrace AI tools, focus on system design, and develop strong communication skills will thrive. The question is not whether AI replaces developers, but whether developers learn to leverage AI faster than the competition does.

**Word count: 198. Time to write: 12-13 min in test conditions.**

### 4.6 Writing scoring + traps

- **Avoid**: SMS-style writing ("u", "thru"), Hinglish words, contractions in formal essay (use "do not" not "don't"), exclamation marks.
- **Lean on**: Connectors ("however", "moreover", "in conclusion"), passive voice for objectivity, varied sentence length.
- **Trap**: Going under 150 words = auto-low score. Over 250 = scorers lose interest. Stay 180-220.

## 5. GenC Next differentiator — what changes

### 5.1 The single biggest difference: coding round difficulty

GenC standard coding problem = Easy (think: palindrome / second largest / frequency count). GenC Next-eligible candidate **same problem solve karta hai, but** Cognizant's algorithm bumps a select pool to a **harder problem in same 45-60 min window** (some cycles), or the same problem is judged with stricter all-test-case-pass + clean-code criteria.

### 5.2 GenC Next-style worked example

> Given an N×N matrix, rotate it 90 degrees clockwise in-place (no extra matrix).

```python
n = int(input())
mat = []
for _ in range(n):
    mat.append(list(map(int, input().split())))

# Step 1: Transpose (swap m[i][j] with m[j][i])
for i in range(n):
    for j in range(i + 1, n):
        mat[i][j], mat[j][i] = mat[j][i], mat[i][j]

# Step 2: Reverse each row
for row in mat:
    row.reverse()

for row in mat:
    print(*row)
```

**Time**: O(n²). **Space**: O(1) — true in-place. Why this is GenC Next territory:
- Two-step transformation logic (not single-pass).
- In-place constraint forbids the simpler "create new matrix" approach.
- Edge cases: 1×1 matrix, 2×2 matrix.

The candidate who recognizes "rotate = transpose + reverse" in 2 min ≠ candidate who builds a quadrant-by-quadrant rotation. Both correct; first is GenC Next.

### 5.3 Composite score targets

- **GenC entry**: ~60% composite, all sections above 50%.
- **GenC Next bracket**: ~75-80% composite, all sections above 65%, coding all test cases pass, communication above-avg.

> **Pro tip**: GenC Next is **not a separate registration**. Tu standard test deta hai, Cognizant tujhe automatically up-bracket karta hai. So **always aim for GenC Next ke metrics** even agar tu standard GenC ke liye apply kar raha hai. Worst case = standard offer.

> **Cross-link**: For exhaustive pseudocode foundations + 30 patterns, see `infosys-sp-playbook` Section 6 (Pseudocode round). Capgemini-specific tilts continue below.

### 10.2 Ten worked Capgemini-style pseudocode examples

#### Example 1: Loop counter (output prediction)

```
i = 1
sum = 0
while i <= 5:
    sum = sum + i * i
    i = i + 1
print sum
```

Trace: i=1 sum=1; i=2 sum=5; i=3 sum=14; i=4 sum=30; i=5 sum=55. **Output: 55**.

#### Example 2: Nested loop counting

```
count = 0
for i = 1 to 4:
    for j = i to 4:
        count = count + 1
print count
```

Inner runs (i=1: 4), (i=2: 3), (i=3: 2), (i=4: 1) = 4+3+2+1 = **10**.

#### Example 3: While with condition

```
n = 100
count = 0
while n > 1:
    n = n / 2
    count = count + 1
print count
```

100 → 50 → 25 → 12 → 6 → 3 → 1, that's 6 divisions. **Output: 6** (assuming integer division). Recognise: this is `floor(log₂(n))`.

#### Example 4: Recursion trace

```
function f(n):
    if n <= 1:
        return n
    return f(n-1) + f(n-2)

print f(5)
```

Fibonacci! f(5) = f(4) + f(3) = 3 + 2 = **5**.

#### Example 5: Bug spotting

```
function reverse(arr, n):                  // Line 1
    for i = 0 to n / 2:                    // Line 2
        temp = arr[i]                       // Line 3
        arr[i] = arr[n - i]                 // Line 4 (BUG)
        arr[n - i] = temp                   // Line 5
```

**Bug on Line 4**: Should be `arr[n - i - 1]` (zero-indexed). As written, it accesses out-of-bounds when i=0 → arr[n] doesn't exist. **Answer: Line 4.**

#### Example 6: Complexity

```
function search(arr, target, low, high):
    if low > high:
        return -1
    mid = (low + high) / 2
    if arr[mid] == target:
        return mid
    if arr[mid] > target:
        return search(arr, target, low, mid - 1)
    return search(arr, target, mid + 1, high)
```

Binary search recursive. **O(log n)** time, **O(log n)** stack space.

#### Example 7: Variable swap trace

```
a = 5
b = 10
a = a + b
b = a - b
a = a - b
print a, b
```

Trace: a=15, b=15-10=5, a=15-5=10. **Output: 10 5** (classic XOR-less swap).

#### Example 8: Array sum with break

```
arr = [3, 1, 4, 1, 5, 9, 2, 6]
sum = 0
for x in arr:
    if x > 5:
        break
    sum = sum + x
print sum
```

Trace: 3, +1=4, +4=8, +1=9, +5=14, then 9 > 5 → break. **Output: 14.**

#### Example 9: String manipulation

```
s = "abcde"
res = ""
for i = length(s) - 1 down to 0:
    res = res + s[i]
print res
```

Reverses string. **Output: "edcba"**.

#### Example 10: Conceptual DS

> Which data structure is best for implementing a function that adds elements and quickly retrieves the *minimum* in O(1)?

(A) Array
(B) Stack
(C) **Min-Heap**
(D) Queue

**Answer: C (Min-Heap)**. Insert is O(log n), retrieve-min is O(1). Stack/Queue are FIFO/LIFO — don't track minimum. Array would need O(n) scan.

## 11. Game-based aptitude — the wild west

### 11.1 The 8 game types

Capgemini's game-based round uses **adaptive Pymetrics-style mini-games** — each ~5-8 minutes, measuring a distinct cognitive trait. The exact set varies by cycle, but commonly seen:

| # | Game name | Measures | Approximate time |
|---|-----------|----------|------------------|
| 1 | **Tower of London** | Planning, sequencing | 7 min |
| 2 | **Numerical Reasoning** | Speed math under pressure | 6 min |
| 3 | **Digit Symbol Substitution** | Working memory + speed | 5 min |
| 4 | **Spatial Reasoning** | 3D rotation, shape mental models | 6 min |
| 5 | **Inductive Reasoning** | Pattern detection in shape sequences | 7 min |
| 6 | **Stop Signal Task** | Inhibition, self-control | 5 min |
| 7 | **Card Sorting** | Cognitive flexibility | 6 min |
| 8 | **Memory N-Back** | Working memory load | 8 min |

**Total ~50 min** for 8 games + transitions = 60-min budget.

### 11.2 The "no-prep" reality — and the workaround

Capgemini explicitly says: "These games measure traits, not knowledge. Preparation cannot help." **Half-true.** What's true: you can't memorise answers; the games are randomised. **What's false**: you can't pattern-practice. Many of these are **direct cousins of standard psychometric tests** that have been around for decades. Doing them once → second time you're 30% faster.

### 11.3 Strategy per game

#### Tower of London
- Goal: rearrange 3 coloured discs across 3 pegs to match a target configuration in **minimum moves**.
- Strategy: **plan moves backward from the target** — visualise the last move, then second-to-last. Don't move blindly.
- Practice resource: search "Tower of London puzzle" online; 50 plays = mastery.

#### Numerical Reasoning
- Speed math: addition / subtraction / multiplication of 2-3 digit numbers in 5 sec each.
- Strategy: **mental arithmetic drill** — `aptitude-quant` Section 2 has Vedic Math shortcuts.

#### Digit Symbol Substitution
- A key shows digit-to-symbol mapping (1→★, 2→●, etc.). You must transcribe sequences fast.
- Strategy: **don't memorise the key**. Glance it each time. Speed > accuracy here in scoring.

#### Spatial Reasoning
- "Which of these shapes is the rotation of this 3D object?"
- Strategy: pick a feature (e.g., "the red dot is on the top-right corner") and track its position across rotations.
- Practice: cubes on YouTube — daily 10-min "mental rotation puzzles" for 2 weeks.

#### Inductive Reasoning
- 4 shape sequences shown, identify the next or odd-one-out.
- Strategy: count features (sides, dots, rotations); look for **arithmetic progressions** or **shape-mapping patterns**.

#### Stop Signal Task
- Press button on green; **don't press** on red. Speed matters.
- Strategy: deliberate **slight hesitation** before responding. ~80 ms hesitation ≈ better inhibition score.

#### Card Sorting
- Sort cards by changing rules (colour → shape → number, with no warning).
- Strategy: **listen for "wrong" feedback** — that's the rule-change signal. Switch criteria ASAP.

#### Memory N-Back
- "Was the current symbol same as N positions ago?"
- Strategy: **rehearse in mental loop**. Don't try to remember exact symbols; remember positions.
- Practice: free apps like "Dual N-Back" — 10 min/day for 2 weeks = noticeable improvement.

### 11.4 Five worked game examples

#### Example 1: Tower of London (mental walkthrough)

> Start: Peg A has [Red, Blue, Green] (bottom to top). Target: Peg C has [Green, Red, Blue].

Solution: Move Green to B. Move Blue to C. Move Red to C. Move Green from B to C. **4 moves.**

#### Example 2: Spatial — cube rotation

> A cube has Red on top, Blue on front, Green on right. After rotating 90° clockwise around the vertical axis (top stays Red), what's on the front?

Front was Blue → moves to right; right (Green) → moves to back; back → moves to left; left → moves to front. So **front = left's old colour**. If left was White → front = White.

#### Example 3: Inductive reasoning — sequence

> Sequence: ▲ ▲▲ ▲▲▲ ___ ?

Each step adds a triangle. Next: **▲▲▲▲ (4 triangles).**

#### Example 4: Numerical reasoning

> A bus travels 240 km in 4 hrs. Same speed, what distance in 5.5 hrs?

Speed = 60 km/h. Distance = 60 × 5.5 = **330 km**.

#### Example 5: Digit symbol substitution

> Key: 1 = ★, 2 = ●, 3 = ◆. Encode sequence: 1, 3, 2, 1, 3, 1.

**Answer: ★ ◆ ● ★ ◆ ★** in <8 sec.

## 12. Coding round (Senior Analyst track) — 3 worked Python solutions

### 12.1 Format details

- **Difficulty mix**: 1 Easy + 2 Medium + 1 Hard typical.
- **Hidden test cases**: 5-10 per problem.
- **Partial credit**: Yes — passing 60% of test cases gets you 60% on that problem.

### 12.2 Three worked Python solutions

#### Problem 1: Maximum Subarray Sum (Kadane's Algorithm)

> Given an array of N integers (can be negative), find the maximum sum of any contiguous subarray.

```python
n = int(input())
arr = list(map(int, input().split()))

current = best = arr[0]
for x in arr[1:]:
    current = max(x, current + x)
    best = max(best, current)

print(best)
```

**Time**: O(n). **Space**: O(1). Capgemini classic — appears in ~25% of Senior Analyst cycles. Edge case: all negatives → return the max single element.

#### Problem 2: Valid Parentheses Check

> Given a string of `()`, `[]`, `{}`, check if all brackets are balanced.

```python
s = input().strip()
pairs = {')': '(', ']': '[', '}': '{'}
stack = []

for ch in s:
    if ch in '([{':
        stack.append(ch)
    elif ch in pairs:
        if not stack or stack.pop() != pairs[ch]:
            print("No")
            exit()

print("Yes" if not stack else "No")
```

**Time**: O(n). **Space**: O(n). Stack-based classic. Capgemini's grading is strict on `[(])` case (which is unbalanced).

#### Problem 3: First Non-Repeating Character

> Given a string, find the first character that appears only once. Return -1 if none.

```python
from collections import Counter

s = input().strip()
c = Counter(s)
for ch in s:
    if c[ch] == 1:
        print(ch)
        break
else:
    print(-1)
```

**Time**: O(n). **Space**: O(k) where k = unique chars. The `for-else` is a Python idiom — `else` runs if no `break` happened. Slick.

- **2 problems in 60 min** = 30 min/problem.
- **Languages**: C, C++, Java, Python.
- **Difficulty mix**: 1 Easy + 1 Medium typical.
- **Hidden test cases**: 5-10 per problem.
- **Partial credit**: Yes — 60% test cases passing = 60% on problem.

### 18.2 Three worked Python solutions

#### Problem 1: Find Missing Number in 1 to N

> Given an array containing N-1 distinct integers from 1 to N, find the missing one. N can be up to 10^6.

```python
n = int(input())
arr = list(map(int, input().split()))

expected = n * (n + 1) // 2
actual = sum(arr)
print(expected - actual)
```

**Time**: O(n). **Space**: O(1). Wipro classic — the math trick (sum formula minus actual sum) avoids O(n) extra space of a set/dict approach.

#### Problem 2: Two-Sum (Hash-Map)

> Given an array of N integers and a target T, find two distinct indices whose values sum to T. Print indices (1-indexed) in ascending order.

```python
n, t = map(int, input().split())
arr = list(map(int, input().split()))

seen = {}
for i, x in enumerate(arr):
    if t - x in seen:
        print(seen[t - x] + 1, i + 1)
        break
    seen[x] = i
else:
    print(-1)
```

**Time**: O(n). **Space**: O(n). The for-else idiom handles the no-match case. Wipro grader expects 1-indexed output (verify problem statement).

#### Problem 3: Longest Substring Without Repeating Characters

> Given a string, find the length of the longest substring with all distinct characters.

```python
s = input().strip()

seen = {}
left = best = 0
for right, ch in enumerate(s):
    if ch in seen and seen[ch] >= left:
        left = seen[ch] + 1
    seen[ch] = right
    best = max(best, right - left + 1)

print(best)
```

**Time**: O(n). **Space**: O(k) where k = unique chars. Sliding-window technique. Medium-difficulty staple in Wipro Elite. Edge case: empty string → 0.

- **Min 0-3**: Read both problems, identify Easy.
- **Min 3-15**: Solve Easy.
- **Min 15-50**: Solve Medium (35 min).
- **Min 50-60**: Test, submit, edge cases.

> Wipro's grader is **lenient on style** — works > pretty. Don't optimise prematurely.
